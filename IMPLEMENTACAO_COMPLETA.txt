@Override
public User updateUser(String name, String pwd, User info) {
    Log.info("updateUser : name = " + name + "; pwd = " + pwd + " ; info = " + info);

    // Check if parameters are valid
    if (name == null || pwd == null || info == null) {
        Log.info("Name, password or info null.");
        throw new WebApplicationException(Status.BAD_REQUEST);
    }

    User user = null;
    try {
        user = hibernate.get(User.class, name);
    } catch(Exception x) {
        x.printStackTrace();
        throw new WebApplicationException(Status.INTERNAL_SERVER_ERROR);
    }

    // Check if user exists and password matches
    if (user == null || !user.getPwd().equals(pwd)) {
        Log.info("User does not exist or password is incorrect.");
        throw new WebApplicationException(Status.FORBIDDEN);
    }

    // Update fields (only if not null in info)
    if (info.getPwd() != null)
        user.setPwd(info.getPwd());
    if (info.getDisplayName() != null)
        user.setDisplayName(info.getDisplayName());
    if (info.getDomain() != null)
        user.setDomain(info.getDomain());
    if (info.getPhoneNumbers() != null)
        user.setPhoneNumbers(info.getPhoneNumbers());

    try {
        hibernate.update(user);
    } catch(Exception x) {
        x.printStackTrace();
        throw new WebApplicationException(Status.INTERNAL_SERVER_ERROR);
    }

    return user;
}

@Override
public List<User> searchUsers(String name, String pwd, String pattern) {
    Log.info("searchUsers : name = " + name + "; pwd = " + pwd + "; pattern = " + pattern);

    // Check if parameters are valid
    if (name == null || pwd == null) {
        Log.info("Name or password null.");
        throw new WebApplicationException(Status.BAD_REQUEST);
    }

    // Verify that the user performing the search exists and has correct password
    User requestingUser = null;
    try {
        requestingUser = hibernate.get(User.class, name);
    } catch(Exception x) {
        x.printStackTrace();
        throw new WebApplicationException(Status.INTERNAL_SERVER_ERROR);
    }

    if (requestingUser == null || !requestingUser.getPwd().equals(pwd)) {
        Log.info("User does not exist or password is incorrect.");
        throw new WebApplicationException(Status.FORBIDDEN);
    }

    // Perform search using JPQL (case-insensitive substring match)
    List<User> results = null;
    try {
        // If pattern is null or empty, return all users
        String searchPattern = (pattern == null || pattern.isEmpty()) ? "" : pattern.toLowerCase();
        String jpqlQuery = "SELECT u FROM User u WHERE LOWER(u.name) LIKE '%" + searchPattern + "%'";
        results = hibernate.jpql(jpqlQuery, User.class);
        
        // Set passwords to empty string as per specification
        for (User u : results) {
            u.setPwd("");
        }
    } catch(Exception x) {
        x.printStackTrace();
        throw new WebApplicationException(Status.INTERNAL_SERVER_ERROR);
    }

    return results;
}

EXPLICAÇÃO:
-----------
✓ Valida parâmetros (name, pwd não podem ser null)
✓ Autentica o utilizador que está a fazer a busca
✓ Executa query JPQL case-insensitive com LOWER()
✓ Pattern é substring: procura "%" + pattern + "%"
✓ Remove passwords dos resultados (segurança)
✓ Se pattern é null/vazio, retorna todos os utilizadores

FLUXO DE VALIDAÇÃO:
-------------------
1. Parâmetros null? → 400 BAD_REQUEST
2. Utilizador não autenticado? → 403 FORBIDDEN
3. Password incorreto? → 403 FORBIDDEN
4. Erro na query? → 500 INTERNAL_SERVER_ERROR
5. Sucesso → 200 OK + Lista de Users (com pwd="")

QUERY JPQL:
-----------
SELECT u FROM User u WHERE LOWER(u.name) LIKE '%pattern%'

Características:
- JPQL (Jakarta Persistence Query Language)
- Case-insensitive: LOWER() em ambos os lados
- Substring match: % antes e depois do pattern
- Retorna lista de objetos User diretamente

protected static final int READ_TIMEOUT = 5000;       // 5 segundos
protected static final int CONNECT_TIMEOUT = 5000;    // 5 segundos

protected static final int RETRY_SLEEP = 5000;        // 5 segundos

SIGNIFICADO:
-----------
READ_TIMEOUT: Tempo máximo para ler resposta após enviar request
CONNECT_TIMEOUT: Tempo máximo para estabelecer conexão TCP
MAX_RETRIES: Número máximo de tentativas em caso de erro transiente
RETRY_SLEEP: Tempo de espera entre tentativas

WORST-CASE SCENARIO:
-------------------
Se todas as 3 tentativas falharem com timeout:
- Tentativa 1: 5s (connect) + 5s (read) = 10s
- Sleep: 5s
- Tentativa 2: 10s
- Sleep: 5s
- Tentativa 3: 10s
TOTAL: 40 segundos até retornar TIMEOUT error

  - Conexão TCP perdida durante transmissão
  - Interface de rede falhou
  - Anomalia temporária de rede (routing)

AÇÃO: RETRY automático
  - Espera RETRY_SLEEP (5000ms)
  - Tenta novamente até MAX_RETRIES (3)
  - Se esgotar tentativas → Result.error(TIMEOUT)

EXCEÇÃO: Exception (genérica)
QUANDO: Erro inesperado (não transiente)
AÇÃO: Log do stack trace, continua loop

IMPORTANTE:
----------
✓ Só ProcessingException triggera retry
✓ Se receber resposta HTTP (mesmo erro 4xx/5xx), NÃO retenta
✓ Apenas erros de comunicação são retentados
